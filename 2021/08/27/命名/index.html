命名

统一资源管理类命名

Bootstrap, Starter

​ 一般来说作为程序启动器使用, 或者是某个项目的基类

Processor

​ 某一类功能的处理器, 用来表示某个处理过程

Manager

​ 对有生命状态的对象进行管理, 通常作为某一类资源的管理入口

Holder

​ 表示持有某个或者某类对象的引用，并可以对其进行统一管理

Factory

​ 工厂模式的命名

Provider

​ 一般是接口或者抽象类, 以便能够完成子实现

Registrar

​ 注册并管理一系列资源

Engine

​ 一般都是核心模块, 用来处理某一类功能

Service

Task

传播类命名

​ 有些参数需要在各种方法中传递, 就需要一个单独的类来统一处理, 最终传递的就是统一的一个类对象

Context

Propagator

回调类命名

Handler, Callback, Trigger, Listener

  • Handler一般都是真正要处理逻辑的对象
  • Callback一般都是一个接口, 用于处理某一类的消息
  • Trigger一般都是某类事件的处理
  • Listener一般用在观察者模式中

Aware

监控类命名

Accumulator

​ 累加器

Metric, Monitor

​ 一般表示监控

Estimator

​ 统计类, 一般指某一类的统计计算

Tracker

​ 跟踪器

内存管理类命名

Allocator

​ 内存分配器

Arena

​ 资源的管理

Chunk

Pool

​ 池化类

过滤处理类命名

Detector

​ 探测器, 一般处理一些事件, 在事件发生时进行捕获或者说响应

Evaluator

​ 一般用于处理计算类

Interceptor

​ 拦截器

Filter

​ 过滤器

Pipeline, Chain

​ 一般在责任链模式中使用

结构类命名

Cache

​ 缓存

Buffer

​ 缓冲

Wrapper

​ 包装, 用于包装某个类做附加处理

Iterator

​ 迭代器

Batch

​ 批处理

Limiter

​ 限流器
